  system:    "You are Deep2Lead's drug discovery AI v2. When given a protein target..."
  user:      "Target: EGFR T790M\nProtein sequence (first 150 AA): ...\nMeasured binding affinity: IC50 = 3.2 nM\nDesign a small molecule drug candidate..."
  assistant: "Rationale: This kinase target presents a conserved ATP-binding hinge region. An N-heterocyclic
              hinge binder combined with a hydrophobic back-pocket group achieves selectivity and nanomolar potency.

🔧 Issues Encountered & Fixes

  • Unsloth model not supported error NotImplementedError: unsloth/gemma-4-E2B-it-unsloth-bnb-4bit is not supported
    Fix: Updated unsloth 2026.4.4 → 2026.5.2 via git install. The bnb-4bit model also requires load_in_4bit=True — using False triggers rejection.
  • Wrong PyTorch wheel for ARM64 (GB10) NotImplementedError: Unsloth cannot find any torch accelerator
    Fix: Standard pip install torch downloads CPU-only aarch64 wheel. Must use:
    pip install torch==2.11.0+cu130 --index-url https://download.pytorch.org/whl/cu130
  • formatting_func returns string, not list ValueError: The formatting_func should return a list of processed strings
    Fix: Pre-apply processor.apply_chat_template() to all 225K records before Dataset.from_dict(), then use dataset_text_field="text". Avoids batch/single ambiguity entirely.
  • Mol-Instructions git-lfs not available
    Fix: git-lfs not installed on dgx1, apt mirror broken. Dataset files are LFS pointers. Solution: Skip Mol-Instructions — BindingDB (150K) + ChEMBL (65K) already gives 225K records, exceeding the 220K target without it.
  • MAMMAL pKd threshold too strict (0.70)
    Fix: Model scores cluster 5.2–5.5 for most inputs (mean pKd=5.79). Official IBM example pair scores 0.549. Lowered threshold to 0.55 in config.py.
  • RDKit SMILES parse error flood
    Fix: Added RDLogger.DisableLog("rdApp.error") + pre-filter regex [0-9\[\]=#@\\/] before any RDKit call so English words (amino, chains) never reach the parser.
