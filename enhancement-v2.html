2 PathoHunt — Browser Game: Overview & Concept

Core Game Concept:

PathoHunt transforms the drug discovery pipeline into an interactive browser game. Pathogens (real proteins with known amino acid sequences) appear as "Boss Enemies." The player's goal is to design a molecule — their "ammunition" — that computationally neutralizes the pathogen. Every game action maps to a real science action: generating candidates with Gemma, scoring with RDKit, and docking with AutoDock Vina.

Design Philosophy

Real Science as Game Engine

No simulated game physics. Every "attack" is a real molecule generation + scoring operation. The game's outcome is 100% determined by actual cheminformatics algorithms. Kids win by doing better chemistry.

Scientific Literacy First

Every damage number comes with a 1-line plain-English explanation. Boss HP is always labeled "Simulation Health." No player ever thinks they cured a disease. They understand they ran a computational model.

Three Engagement Levels

Casual (Quick Battle) — 2D mode, 5 min max, guided hints. Intermediate (Docking Battle) — 3D Vina docking, pick your strategy. Advanced (AI Army) — autonomous loop with live visualization.

Reusable for Class & Solo

Designed for both individual play and classroom demos. The AI Army mode is perfect for group viewing on a projector — watch the AI optimize a drug molecule in real time.

Navigation Entry Point

Left Sidebar → "Game" (new menu item). Icon: 🎮. Appears between "Auto Experiment" and "Missions" in the sidebar. Clicking goes to the Game Lobby (/game route). No auth restriction — logged-in students, admins, and faculty can all play. Guest access can be enabled by admin for demos.

5 Game Mechanics, Scoring & Progression

Boss HP Calculation

Boss HP starts at 100% (representing the known drug's baseline score). Each player attack (molecule submission) potentially reduces HP.

# Boss HP = normalized composite score of the CURRENT BEST molecule in this session
# Starts at HP_initial = normalize(known_drug_score)

def calculate_boss_hp(session: GameSession) -> float:
    if not session.attacks:
        return 100.0  # max HP before any attack

    best_attack = max(session.attacks, key=lambda a: a.composite_score)

    # HP = inverse of player's best composite score
    # Higher player score → lower boss HP
    player_best = best_attack.composite_score   # 0.0 → 1.0
    boss_hp = max(0.0, 100.0 - (player_best * 100.0))
    return round(boss_hp, 1)

# Boss is defeated when HP ≤ 10.0 (player score ≥ 0.9)
# Or for Docking mode: when docking_score ≤ win_threshold (per target)
Educational framing: HP does not mean "virus health" in any real sense. It is explicitly labeled "Simulation Score" and the disclaimer is shown on every battle screen.

New Service: services/game_service.py

class GameService:

    def start_session(user_id, target_id, mode, difficulty) -> GameSession:
        """Create session, set boss HP from known drug baseline, return session_id"""
        target = target_service.get_curated_target(target_id)
        known_score = target['known_drug_score']   # pre-set in curated_targets.json
        boss_hp = 100.0 - (known_score * 100.0)    # e.g., 0.72 score → 28% HP already gone
        session = GameSession(user_id=user_id, target=target, mode=mode, ...)
        ...

    def execute_attack(session_id, smiles, user_id) -> AttackResult:
        """Generate molecules if needed, score/dock, calculate damage, update HP"""
        session = get_session(session_id)
        if session.mode == 'quick_battle':
            candidates = molecule_generator.generate(smiles, ...)
            scored = [property_calculator.compute_all(c) for c in candidates]
            best = max(scored, key=lambda x: x['composite_score'])
        elif session.mode == 'docking_battle':
            # reuse existing docking_service pipeline
            best = docking_service.run_docking_pipeline(smiles, session.pdb_path, ...)

        damage = calculate_damage(best, session.previous_best)
        new_hp = max(0.0, session.boss_current_hp - damage)

        attack = GameAttack(session_id=session_id, smiles=best.smiles, ...)
        session.boss_current_hp = new_hp

        if new_hp <= 0 or best.composite_score >= win_threshold[session.difficulty]:
            session.status = 'won'
            xp_service.award_xp(user_id, f'game_boss_defeated_{difficulty}')

        return AttackResult(damage=damage, new_hp=new_hp, best_candidate=best, ...)

    def calculate_damage(current_best, previous_best) -> float:
        """Map score improvement to damage points"""
        delta = current_best['composite_score'] - (previous_best or 0.0)
        if delta <= 0:
            return 0.0  # no damage if no improvement
        damage = delta * 100.0
        # bonus points for special properties
        if current_best.get('novelty_status') == 'novel':
            damage += 15.0
        if current_best.get('lipinski_pass'):
            damage += 10.0
        return min(damage, 60.0)  # cap per-attack damage

New DB Models: models/game_models.py

class GameSession(db.Model):
    __tablename__ = 'game_session'
    id                = Column(String(36), primary_key=True, default=lambda: str(uuid4()))
    user_id           = Column(String(36), ForeignKey('users.id'), nullable=False)
    target_id         = Column(String(64), nullable=False)   # curated target slug
    mode              = Column(Enum('quick_battle','docking_battle','ai_army'), nullable=False)
    difficulty        = Column(Enum('junior','fellow','pi','nobel'), default='junior')
    status            = Column(Enum('active','won','lost','abandoned'), default='active')
    boss_initial_hp   = Column(Float, nullable=False)
    boss_current_hp   = Column(Float, nullable=False)
    attacks_count     = Column(Integer, default=0)
    best_score        = Column(Float, default=0.0)
    win_threshold     = Column(Float, nullable=False)
    time_started      = Column(DateTime, default=datetime.utcnow)
    time_ended        = Column(DateTime, nullable=True)
    attacks           = relationship('GameAttack', back_populates='session', lazy='dynamic')

class GameAttack(db.Model):
    __tablename__ = 'game_attack'
    id              = Column(String(36), primary_key=True, default=lambda: str(uuid4()))
    session_id      = Column(String(36), ForeignKey('game_session.id'), nullable=False)
    smiles          = Column(Text, nullable=False)
    composite_score = Column(Float, nullable=False)
    docking_score   = Column(Float, nullable=True)    # kcal/mol, 3D mode only
    damage_dealt    = Column(Float, default=0.0)
    boss_hp_after   = Column(Float, nullable=False)
    novelty_status  = Column(String(16), nullable=True)
    lipinski_pass   = Column(Boolean, default=False)
    is_best         = Column(Boolean, default=False)
    created_at      = Column(DateTime, default=datetime.utcnow)
    session         = relationship('GameSession', back_populates='attacks')

9 Max Jaderberg Framework — Applied to Deep2Lead

Key insights from the Max Jaderberg drug discovery conversation, mapped to actionable decisions for Deep2Lead v3 architecture.

1Move Beyond Local Models — Build Foundation-Style, Cross-Target Generalization

"Avoid focusing on models that fit specific local data for a single target. Aim to build general, foundation-style models that can be applied across the entire protein universe without constant fine-tuning."

Deep2Lead Application: The current Gemma4 molecule generator uses a fixed prompt template per experiment — a "local" approach. The fine-tuning plan (see finetune-gemma.html) addresses this by training on cross-target datasets (ChEMBL, BindingDB) so that Gemma generates better molecules regardless of which pathogen is targeted. The game's 8 different bosses naturally push students to think cross-target.

2Half a Dozen Hard Problems — What Deep2Lead Must Solve

"Solving ~6 fundamental biological/biochemical problems with AlphaFold-level accuracy creates a transformative drug design engine."

Deep2Lead Current Coverage:

ProblemStatus in Deep2Lead
① Protein structure predictionSolved (AlphaFold EBI + ESMFold)
② Binding site predictionPartial (manual binding site coords in JSON)
③ Molecular docking / binding affinitySolved (AutoDock Vina)
