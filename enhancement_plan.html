1 Product Overview & Vision

Deep2Lead v3 transforms the existing Georgia Tech graduate tool into a hands-on educational AI drug discovery platform for high school students. The platform lets students move from protein sequences and SMILES strings through 3D structure prediction, molecular docking, AI-generated candidate molecules, novelty checking, and responsible scientific interpretation.

Core Educational Message:
"AI can help scientists explore ideas faster — but every computational prediction must eventually be validated through real experiments, safety testing, and clinical studies."

Target Users

  • High school students (grades 9–12) interested in AI, biology, chemistry, medicine
  • Summer STEM program participants
  • Secondary: parents, teachers, college applicants seeking research exposure

Lock-and-Key Analogy (runs throughout the app)

  • Protein target = the lock (3D shape determines what fits)
  • Drug molecule = the key (must fit the binding pocket)
  • Structure prediction = turning the sequence instructions into a 3D lock
  • Molecular docking = testing whether the key fits and how tightly
  • AI generation = a robot designing thousands of new key shapes overnight

AutoDock Vina Integration

# pip install vina meeko rdkit openbabel-wheel
# New service: v2/services/docking_service.py

from rdkit import Chem
from rdkit.Chem import AllChem
from meeko import MoleculePreparation
from vina import Vina
import subprocess, tempfile, os

def prepare_ligand_pdbqt(smiles: str) -> str:
    mol = Chem.AddHs(Chem.MolFromSmiles(smiles))
    AllChem.EmbedMolecule(mol, AllChem.ETKDGv3())
    prep = MoleculePreparation()
    prep.prepare(mol)
    return prep.write_pdbqt_string()

def prepare_receptor_pdbqt(pdb_path: str) -> str:
    out_path = pdb_path.replace(".pdb", ".pdbqt")
    subprocess.run(
        ["obabel", pdb_path, "-O", out_path, "-xr", "-p", "7.4"],
        check=True, capture_output=True
    )
    return out_path

def dock_molecule(receptor_pdbqt: str, ligand_pdbqt_str: str,
                  center: list, box_size: list = [25, 25, 25],
                  exhaustiveness: int = 8) -> dict:
    v = Vina(sf_name="vina", verbosity=0)
    v.set_receptor(receptor_pdbqt)
    v.set_ligand_from_string(ligand_pdbqt_str)
    v.compute_vina_maps(center=center, box_size=box_size)
    v.dock(exhaustiveness=exhaustiveness, n_poses=5)
    score = v.score()[0]   # kcal/mol, more negative = better
    return {"docking_score_kcal": score, "poses_pdbqt": v.poses(n_poses=1)}

6 Auto Experiment — Autonomous Optimization Loop

Inspired by Karpathy's autoresearch project: an AI agent autonomously iterates on a single editable surface (there: train.py; here: molecule generation prompt + parameters) and keeps changes only when the objective metric improves.

The drug discovery equivalent: Generate → Validate → Dock → Score → Keep best → Seed next round.

Auto Experiment Loop

🎯
Set Target & Baseline
Select protein + seed SMILES. Run round 0.
🧪
Generate Candidates
Gemma4 LLM generates N SMILES from current best seed.
✅
Validate
RDKit: syntax, valency, MW, Lipinski filters.
⚗️
Prepare & Dock
Meeko → PDBQT. AutoDock Vina. Get kcal/mol score.
📊
Score
Composite = docking + QED + novelty + SAS + Lipinski.
🏆
Keep or Discard
If improved → keep as next seed. Else → revert.
🔁
Repeat
Run N rounds or until budget reached.
📝
Report
Show round-by-round score trend + best molecule.

Key Design Decisions (from autoresearch)

Fixed Compute Budget

Each round has a max time limit (e.g., 5 min per round). No unbounded loops. Students set max rounds (2–10) and molecules per round (5–20). Budget shown as progress bar.

Single Editable Surface

Only the Gemma4 prompt + generation parameters (temperature, noise_level, similarity_constraint) change between rounds, not the scoring logic. Diffs are reviewable.

Transparent Logging

Every round logged: iteration, seed SMILES, best score, status (keep/discard), short rationale. Students see the reasoning, not just the result.

Direction: Higher is Better

Composite score goes from 0→1. More negative docking kcal/mol = better. App normalizes docking to [0,1] using sigmoid. Students see one unified "mission score".

New Flask Blueprint: api/auto_experiment.py

POST /api/v3/auto-experiment/run          → start async loop (Celery task)
GET  /api/v3/auto-experiment/{id}/status  → SSE stream of round-by-round progress
POST /api/v3/auto-experiment/{id}/stop    → stop gracefully after current round
GET  /api/v3/auto-experiment/{id}/report  → final ranked list + score trend JSON
Async approach: Docking jobs take 10–120s each. Use Python threading.Thread (same pattern as autoresearch's AutoResearchRunner) with Server-Sent Events (SSE) to stream round updates to the frontend without blocking gunicorn. No Celery needed for MVP.

Boss Health Bar Concept

The protein target is presented as a "challenge boss". The boss health bar represents the simulation's binding score.

Critical wording rule: Always say "simulation health" or "model score." Never say "you cured the disease" or "you destroyed the virus." Example text: "Your molecule reduced the target's simulation health by 62%. This means your candidate scored better in the computational model. It does not mean the molecule is a real medicine."

11 Safety & Educational Guardrails

Language Rules

  • ✅ "simulation score" / "model prediction" / "computational estimate"
  • ✅ "candidate molecule" / "educational experiment"
  • ❌ "cure" / "destroy virus" / "real medicine" / "approved drug"
  • ❌ Any claim that a high score means a molecule is a real drug

Compute Budget Limits

  • Max 10 candidates per round (beginner), 20 (advanced)
  • Max 5 rounds per Auto Experiment
  • Docking exhaustiveness capped at 16 per candidate
  • ESMFold: max 400 AA, 1 fold per experiment per student
  • Rate limiting: 10 docking jobs / hour per user

Required Disclaimers

  • Every experiment results page: "These are computer simulation results only."
  • Docking scores always shown with: "Lower kcal/mol = better simulated fit — not proof of real binding."
  • Before Auto Experiment starts: explanation of what each round means
  • Boss health bar always shows the disclaimer sentence

Target Library Curation

  • No dual-use pathogen targets (no enhanced-pathogen work)
  • Only use well-characterized, publicly known targets
  • Advanced mode (custom sequence) requires admin unlock by teacher
  • Molecule generation prompt includes safety filter in system prompt
Summary in one sentence:

Deep2Lead v3 takes a working graduate drug-discovery tool and rebuilds it as a mission-driven, 3D-enabled, autonomously-looping educational game — where every score comes with an explanation, every claim comes with a disclaimer, and every student leaves understanding that real drug discovery is a marathon, not a keyboard shortcut.
