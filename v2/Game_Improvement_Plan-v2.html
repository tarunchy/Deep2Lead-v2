Design Document · PathoHunt 3D

Game Improvement Plan v2

Date: 2026-05-11 Platform: Deep2Lead v2 Engine: Three.js r128 + Flask + Gemma4 LLM Research: Foldit, Hades, Dead Cells, FTL, DrugHunter

PathoHunt 3D is a browser-based arcade game where players fire AI-generated drug molecules at real pathogen boss proteins. This document captures research-backed improvement ideas — from quick UI wins to deep gameplay overhauls — grounded in what makes scientific games like Foldit sticky, and what makes roguelites like Hades endlessly replayable.

✓
What's Already Built
The current feature set as of May 2026

⚔️ 3D Boss Fights

7 real protein targets (COVID Mpro, EGFR, HIV Protease…) with Three.js meshes, HP bars, 300 HP per boss.

🧬 AI Molecule Deck

Gemma4 LLM generates 5 molecules per attack round. Composite score (QED + SAS + Tanimoto) drives damage.

🎮 Ship Controls

Arrow-key ship movement, spacebar fire, missile type + trajectory selectors, parabolic arc support.

💀 Boss Dodge System

20/35/55% dodge chance by difficulty, evasion mode, +15% wounded. TTS voice taunt on miss.

🟢 Friendly Fire

Green safe-cell obstacles; hitting them drains player HP + voice alert + centred popup.

🔊 Kokoro TTS

Professor AI narrates attacks, evades, friendly-fire, low-HP warnings, tutorial steps.

🔥 Combo System

3+ consecutive hits ≥55% trigger Hot Combo badge + 1.25x damage multiplier.

📖 Story Screen

Mission briefing overlay with boss lore before each battle starts.

🎓 Tutorial

8-step narrated tutorial with Kokoro TTS, animated visuals, progress dots.

🎯 Game History

Full session log — battles, molecules, boss HP drained — with save-to-experiments.

📂 Save to Experiments

Any game session's best molecules can be saved as a draft Experiment for further analysis.

🏅 XP & Badges

XP events for game start, first attack, boss damaged, boss defeated. Boss-specific badges.

Quick Wins — High Impact, Low Effort
Can be shipped in 1–3 days each. Strong ROI on engagement.
🧑‍⚕️
Patient Story Narrative Cards Emotional stakes that make QED scores feel real
High Priority Low Effort
Before each boss encounter, display a 3-panel narrative: a patient vignette (e.g., "Maria, 42, diagnosed with EGFR+ lung cancer — her tumor has developed resistance to erlotinib"), the biological villain (the target protein), and a cliffhanger. After victory, close the loop with an outcome card. Pipe through existing Kokoro TTS.
Why it works: Foldit's most successful campaigns (HIV protease, flu) were framed around real patients. Emotional stakes convert casual players into committed ones. QED stops being an abstract number — it becomes the drug's chance of helping Maria.
// Implementation Add `patient_story` and `victory_outcome` fields to game_levels.json per boss. Extend the existing story screen in game_battle_3d.html with a 3-step card carousel. TTS narration already wired.
🔬
Contextual Science Flash Cards on Hit Turn every score into a micro-learning moment
High Priority Low Effort
After each attack, the existing Science Card fires generically. Make it score-aware: if QED is high → explain drug-likeness with a real example; if SAS is bad → show a failed drug with poor synthesizability; if Tanimoto is high against a known drug → name that drug and its clinical status. 2–3 sentence overlays, dismissible.
Why it works: Kerbal Space Program retains players because every failure teaches real physics. PathoHunt's equivalent is cheminformatics — every score tells a real story. Contextual > generic.
// Implementation Add score-branch logic to showScienceCard() in game_battle_3d.js. Create a scoreContextMap object with thresholds → explanations for QED, SAS, tanimoto. No backend changes needed.
🛡️
Safety Net — Guaranteed Strong Card After 3 Bad Attacks Frustration prevention without removing skill ceiling
High Priority Low Effort
After 3 consecutive attacks with composite < 0.40, the server guarantees the next deck includes at least one molecule with composite ≥ 0.60. Tracked via a bad_streak counter on the session. Resets on any hit ≥ 0.40.
Why it works: The most common cause of early abandonment is a string of weak LLM outputs. Dead Cells uses a "pity timer" system for item drops — same principle. Players feel unlucky, not incompetent.
// Implementation Add bad_streak to GameSession model (or track in execute_attack locally). In execute_attack(), if bad_streak >= 3: reseed generator with known_drug_smiles from game_levels.json. Reset streak counter after a decent hit.
📊
Post-Run Researcher's Notebook Make players feel like scientists, not just gamers
High Priority Low Effort
After every run (win or loss), show a summary screen with a scatter plot: QED on X-axis, composite score on Y-axis, all molecules fired color-coded by result. A "Download CSV" button exports SMILES + scores. An "Add to Experiments" button (already built) is prominently featured.
Why it works: Foldit made players feel like scientists because their data was real. PathoHunt generates genuinely novel AI-designed molecules — players should feel the weight of that. The CSV export also makes it a lightweight real drug discovery tool.
// Implementation Add a post-run overlay to game_battle_3d.html (after onVictory and game-over screens). Use Chart.js (already a viable CDN add) for the scatter plot. Data is already available via session.attacks. CSV download is a client-side Blob export.

Strong engagement multipliers. Worth the investment.

Phased Boss Fight with Mechanic Shifts Stop the "hit spacebar until dead" loop
Medium Priority Medium Effort
Divide each boss's HP bar into 3 phases (100–60%, 60–30%, <30%). Each phase shift triggers: a visual Three.js mesh transformation, a new mechanic (phase 2 = mutant safe cells spawn faster; phase 3 = boss fires back with spores), and a TTS voice line from the boss ("You only know one binding mode? I've mutated!").
Why it works: FTL's multi-phase boss encounters prevent the repetitive loop that currently exists. Boss phase variance is already scaffolded in the code via idleAnim types — phased behavior extends that naturally.
// Implementation Add phase tracking to PathoHunt3D class. On HP crossing 60% and 30%, call enterPhase(n). enterPhase() changes bossProfile.idleAnim, increases spore spawn rate, triggers TTS. Phase transitions logged to GameSession for history display.
🏆
Global Per-Boss Leaderboard Competitive pressure drives better molecular designs
Medium Priority Medium Effort
After each victory, the winning SMILES and composite score are posted to a global per-boss leaderboard. Entries display the player's handle, molecule codename (already generated: CX-4231), composite score, and a "novelty badge" (ChEMBL validation already exists). Top 3 get a "World Record Breaker" badge. Leaderboard shows SMILES so others can learn from top entries.
Why it works: Foldit's leaderboards produced a Nature paper (2011) because competition drove better designs. The SMILES leaderboard turns it into a crowdsourced SAR table — players learn from each other's chemistry.
// Implementation New DB table: GameLeaderboardEntry (target_id, user_id, smiles, composite_score, attack_count, won_at). New route: GET /api/v3/game/leaderboard/{target_id}. Update execute_attack() to insert entry on win. New leaderboard panel on game selector page.
📌
Scaffold Pinning — Player Steers the LLM Give players creative agency over chemistry
Medium Priority Medium Effort
Allow players to pin a scaffold fragment from any card in the deck with one click. The next LLM generation is seeded with that pinned fragment, steering exploration. Display the pinned fragment as a small structural token above the deck with a ✕ to clear. Teaches Structure-Activity Relationships (SAR) through play.
Why it works: Currently the deck is entirely LLM-driven with no player creative agency between shots. Foldit's core loop is iterative manual optimization — pinning gives PathoHunt an equivalent. Players make a real med-chem decision (scaffold retention) with a single click.
// Implementation Add "📌 Pin" button to each mol-card. Store this.pinnedSmiles in PathoHunt3D state. Pass pinnedSmiles as seed_bias to /candidates API. In generate() in molecule_generator.py: if seed_bias provided, prepend it to the LLM prompt as "retain this scaffold: {smiles}".
⏱️
Outbreak Mode — Timed Urgency Run Time pressure makes the science feel urgent
Medium Priority Medium Effort
A special run mode with an 8-minute countdown replacing the HP-based loss condition. Narrative: a pathogen is spreading through a city — each minute represents one hospital overwhelmed. Background darkens progressively, spore spawn rate escalates automatically, a city silhouette "fills with red" in the HUD corner.
Why it works: Time pressure is the oldest engagement lever in arcade design (Space Invaders, 1978). Outbreak framing — used effectively in Foldit's HIV campaign — makes urgency visceral and the science personally relevant.
// Implementation Add outbreak_mode boolean to GameSession. Client tracks countdown timer in animate(). Gradually increase DIFFICULTY.normal.spawnRate as time passes (every 2 min: spawnRate -= 500). Add HUD countdown and city health bar elements. Payout: 2x Research Points (future meta-progression).

Meta-Progression — Lab Upgrade System Make losing feel productive
Future High Effort
Add a persistent "Research Points" currency earned from every run (win or lose). Research Points unlock permanent account-level upgrades: faster LLM deck generation, a "Scaffold Library" that biases LLM toward known chemotypes, +1 molecule card slot, a "Boost" that doubles QED weight for one attack.
Why it works: Hades proved that losing should feel productive. Currently losing PathoHunt is a dead end. Meta-progression transforms failed runs into stepping stones. Unlocked scaffolds also give players a stronger starting hand on weak LLM days.
// Implementation New DB table: UserResearchPoints, LabUpgrade (slug, name, cost, effect_type, effect_value), UserLabUpgrade. New /lab-upgrades page. Integrate with existing XP/badge system in gamification.py. Client-side: apply active upgrades when rendering deck and computing damage.
🦠
Boss Mutation Events — Real Resistance Biology Players encounter real drug resistance through play
Future High Effort
At each phase transition, the boss gains a random biologically real resistance mechanism: "Efflux Pump Activated" (reduces damage from molecules below MW 400 by 50%), "Binding Site Mutation" (Tanimoto penalized instead of rewarded), or "Protein Overexpression" (HP regenerates 2 pts/sec). Each mutation shows a brief real-world explanation.
Why it works: Hades' Olympian Boons require strategic adaptation. Critically, each mutation corresponds to a real clinical phenomenon — players learn genuine drug-resistance biology through play, not lectures.
// Implementation Define MUTATION_POOL in game_levels.json per boss (biologically accurate resistance types). Apply mutation effects in _calculate_damage() server-side using session.active_mutation. Client shows mutation alert overlay at phase transition.
👥
Co-op Research Team Mode Two-player async: Attacker + Lab Analyst
Future High Effort
Two logged-in users join the same boss session. Player 1 fires offensive molecules (existing mechanic). Player 2 plays "Lab Analyst": manually annotates molecules in the log (thumbs up/down), which biases the next LLM deck for Player 1. Shared HP bar, shared Research Points, joint victory screen.
Why it works: Phylo (McGill RNA-folding game) found co-op sessions produced 3× higher quality solutions. The analyst role maps to a real med-chem workflow (chemists annotate SAR for computational colleagues). Async-friendly — no real-time sync required.
// Implementation Add co_player_id and analyst_annotations (JSON) to GameSession. New route: POST /api/v3/game/session/{id}/annotate. In execute_attack(): weight annotated molecules higher in LLM seed selection. Invite flow via shared session link.
🧠
Molecular Memory — Boss Learns from All Players Community discovery drives genuine structural novelty
Future High Effort
Maintain a per-boss history of top 20 SMILES across all players. When the boss generates a dodge event, preferentially dodge molecules structurally similar (Tanimoto > 0.6) to previously submitted ones. New players face a boss that has already "seen" prior attacks — encouraging genuine structural novelty. Makes ChEMBL novelty validation mechanically consequential.
Why it works: The community collectively discovers what works, forcing everyone to evolve. A truly novel molecule bypasses the boss's memory and hits harder — scientifically modelling acquired drug resistance.
// Implementation New table: BossMemory (target_id, smiles, tanimoto_fingerprint, frequency). On dodge check: compute Tanimoto vs memory pool. If Tanimoto > 0.6 to any memory entry: dodge_chance += 0.20. Update memory pool on each won session (top 3 molecules).
