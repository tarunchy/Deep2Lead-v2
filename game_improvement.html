Step 01
Current State Diagnosis
What Gemini built is visually impressive but functionally broken in 10 key areas. These are not cosmetic — they affect whether the game is actually playable and educational.
🚫
No pathogen selection screen for 3D game Critical
URL goes directly to /game/pathohunt-3d/covid19_mpro. There is no lobby/selector for 3D mode. The game lobby (/game) still points to the old 2D battle route. Kids have no idea how to discover or choose a monster to fight.
🐛
All bosses use identical visual — red spinning torus knot Critical
Regardless of which pathogen is selected (Flu, COVID, HIV, cancer), the monster looks exactly the same. The game_battle_3d.js has no per-boss logic. A kid fighting "Flu Commander" sees the same monster as "CDK2 Overlord". No narrative identity.
🐛
Attack result is async-disconnected from visual Critical
Projectile hits monster visually immediately, but the API call resolves later. The code sets proj.apiResult asynchronously, so often the hit registers with no apiResult and only subtracts 2 HP regardless of molecule quality. Players see no feedback that their choice matters.
🚫
Molecule spells are 5 hardcoded known drugs, not AI-generated Missing Feature
The SPELL_DATABASE in JS has Aspirin, Caffeine, Ibuprofen, Sulfanilamide, Salicylic Acid. The entire point of the game is AI-powered drug design, but the player never sees AI-generated molecules. The generate() service runs on the server but results are silently discarded.
🐛
SMILES input is hidden — player has no agency UX
The SMILES field is <input type="hidden">. The spell display only shows a preset name. Players cannot enter their own molecules, cannot see what SMILES string they are firing. The "CLICK ARENA TO FIRE" hint suggests interaction, but there's no way for a player to deliberately design anything.
🎨
Zero science feedback after each attack Missing Feature
When an attack lands, no score breakdown is shown. The API returns QED, SAS, Lipinski, composite score, damage — all are ignored by the frontend. Kids never learn why one molecule is better than another. This is the core educational hook that's completely absent.
🎨
No game intro / boss story shown before battle UX
The game_levels.json has rich lore: boss_flavor, game_intro, and plain_english fields. None of this is shown in the 3D game. A kid entering the COVID boss gets no context about what they're fighting or why it matters.
🐛
Cross-Validate is completely fake Critical
The crossValidate() function in JS uses setTimeout(2000) and checks tanimoto < 0.8 against a hardcoded value. It shows "NOVEL CANDIDATE DETECTED" or "EXISTING ANALOG" based on nothing real. The existing /api/v3/enrich endpoint is never called.
🚫
Tutorial hardcodes COVID boss, skips lobby UX
The "Skip Training & Join Battle" button in game_tutorial_3d.html hardcodes /game/pathohunt-3d/covid19_mpro. Tutorial final step also redirects there. Players who want to fight a different boss have no path.
🎨
No session summary or "best molecule" highlight on victory Missing Feature
The victory screen shows only the final SMILES. There's no summary of how many attempts were made, score progression chart, or comparison of best molecule vs the known drug. The XP and badge are awarded server-side but never displayed in the 3D game.

Step 03
Five Improvement Phases
Each phase is self-contained and can be reviewed independently. UI/UX appearance is preserved — only functionality, data connections, and missing components are added.
P1
Pathogen Monster Selector — The Missing Entry Point
New page: /game/pathohunt-3d (no target_id) · Files: game.py, game_lobby.html, new template

The 3D game has no front door. This phase adds a monster selection screen styled like a sci-fi threat board — identical concept to the /run-3d "Step 1: Target" picker, but with each pathogen displayed as a unique animated 3D monster card. The game lobby will also be updated to route to 3D.

  • New route: GET /game/pathohunt-3d
    Returns a new template showing all bosses in a grid. Clicking a boss enters /game/pathohunt-3d/<id>. Add to api/game.py.
  • Monster cards with Three.js mini-previews
    Each boss card has a small inline Three.js canvas (~120×120px) rendering its unique 3D shape. On hover the shape spins faster. Clicking expands a detail overlay with boss lore.
  • Boss detail overlay with TTS intro
    When a boss card is clicked: slide-in overlay shows boss_flavor text, plain_english description, difficulty badge, and a "ENTER BATTLE" button. TTS narrates the game_intro text via /api/v3/game/tts.
  • Update game lobby to route 3D
    Change boss card hrefs in game_lobby.html from /game/battle/<id> to /game/pathohunt-3d/<id>. Add "3D ARENA" mode badge to each card.
  • Tutorial "Skip" button → selector
    Change tutorial's hardcoded /game/pathohunt-3d/covid19_mpro links to /game/pathohunt-3d so players choose their boss.

P2
Unique Monster Visuals Per Pathogen
Files: game_battle_3d.js · Boss shape + color driven by target_id

Currently all bosses are the same red TorusKnot. Each pathogen in biology has a distinct visual character — the flu virus looks different from HIV which looks different from a kinase. We map each target to a unique Three.js geometry, color palette, and animation behavior.

  • BOSS_PROFILES map in game_battle_3d.js
    A JS constant object keyed by target_id containing: geometry constructor, primary color, emissive color, scale, rotation speed, and a buildMesh() factory method. Replaces the current hardcoded TorusKnot.
  • Boss idle animation per type
    Flu: wobble/pulse. Corona: slow rotation with spike jitter. HIV: multi-head orbit. Kinase: ring spin. BRAF: erratic twitch (mutation behavior). SIRT1: gentle float. CDK2: mechanical step rotation.
  • Boss "wounded" visual state
    When HP drops below 50%, monster geometry starts fragmenting (vertices offset). Below 25%, it glitches and flickers. This gives visual progression without changing the core render loop.
  • Spore color matches boss type
    Incoming obstacle spores take the boss's primary color. When a spore is destroyed, the explosion color matches the molecule (cyan for standard, magenta for hypersonic). Creates visual connection between pathogen and molecule.
P3
AI Molecule Deck — Fix the Core Attack Loop
Files: game_battle_3d.js, game_battle_3d.html · This is the biggest functional fix

The entire point of PathoHunt is AI-powered drug design. The server already generates molecules via Gemma4 and scores them — but the frontend throws this data away. This phase surfaces the AI candidates as a visual "molecule deck" and fixes the async attack flow so damage is accurately applied.

Before
5 hardcoded preset molecules (Aspirin etc.)
Player cycles through ◀ ▶ arrows, no choice
Projectile hits monster BEFORE API responds
If API slow: always -2 HP regardless of score
Generated molecules silently discarded
SMILES hidden, player sees only preset name
After
3 AI-generated molecule cards shown per round
Player taps a card to select, then clicks to fire
Projectile held in "pending" state at monster
Impact applies AFTER API result (with spinner)
Card shows molecule preview + estimated power
SMILES visible; "Custom SMILES" input option
  • Molecule deck UI — replaces spell console
    Bottom HUD shows 3 cards horizontally. Each card: molecule code name (auto-generated e.g. "CX-7142"), a short SMILES truncated to 20 chars, and colored power indicator (green/yellow/red estimated from molecular weight). Cards refresh after each attack.
  • Fetch initial deck from /api/v3/game/session/<id>/candidates
    New lightweight endpoint: calls molecule_generator.generate() with the boss's starter SMILES, returns 3 candidates with basic metadata. Called on session start and after each attack. No heavy scoring — just generation.
  • Fix async attack: "pending impact" state
    When projectile reaches monster, instead of immediately applying damage: play "ANALYZING" animation on monster (pulse glow), lock further attacks, await API response, then apply damage with a visible floating number. Timeout fallback: if API takes >8s, apply -1 HP and unlock.
  • Custom SMILES input mode
    A "⌨️ Custom" button on the deck opens a text input for advanced players to enter their own SMILES. Validates client-side (basic character check) before enabling fire. For kids: hide by default behind "Expert Mode" toggle in tactical sidebar.
  • Attack count indicator
    Show remaining attacks (e.g. "Attack 3/10") for difficulty modes with limits. In Junior mode show "∞ attempts" with a soft encouragement message instead of pressure.

P4
Science Feedback Cards — The Education Layer
Files: game_battle_3d.js, game_battle_3d.html · Kid-friendly science after every attack

PathoHunt is designed for kids and non-technical users. After every attack the game should explain in plain English what happened and why the molecule scored as it did. This is the core educational value — transforming raw computational scores into a story a 10-year-old can follow.

  • Post-attack science card panel
    A slide-in card appears on the left after each hit. Shows: molecule name, 3 simple stats with emoji ratings (💊 Drugability, 🔬 Ease to make, 🎯 Target fit), one plain-English sentence about what happened. Auto-dismisses after 5s or on next click. Never blocks the arena.
  • Kid-friendly score translation
    QED → "Drugability ⭐⭐⭐⭐" (0-5 stars). SAS → "Hard to Make / Easy to Make". Lipinski pass → "Body-Friendly ✅". Composite → "POWER LEVEL: 74%". All technical terms hidden behind a "Show Science Details" toggle for advanced users.
  • TTS narration on score result
    After each attack result, a 1-sentence TTS narration plays: e.g. "Great hit! Your molecule blocked 18% of the virus's scissors. Try making it even more body-friendly next round!" Uses existing /api/v3/game/tts. Can be muted from HUD.
  • Combat log shows real score data
    Update the bottom-left combat log to show: > ATTACK 3: Score 0.74 | Damage -18.5 HP | NEW BEST! instead of the current generic messages. Color-code: green for new best, yellow for improvement, dim for no improvement.
  • Score progression indicator
    Add a thin "BEST SCORE" bar under the boss HP bar. It fills toward the win threshold. A gold star marker shows where the win threshold is. This gives players a clear visual goal independent of HP.
  • "Kids Mode" / "Science Mode" toggle
    In the tactical sidebar, add a mode toggle. Kids Mode: all scores as emoji stars + plain English + TTS always on. Science Mode: raw QED/SAS/Tanimoto values + SMILES editable. Default: Kids Mode. Preference saved in localStorage.

P5
Real Cross-Validate + Meaningful Victory Screen
Files: game_battle_3d.js, game_battle_3d.html, api/game.py

The current victory screen is nearly empty and cross-validate is fake. This phase transforms victory into a genuine moment of discovery — showing the player's best molecule compared to the known drug, and actually checking if it's novel.

  • Fix cross-validate: call real enrich API
    Replace the fake setTimeout in crossValidate() with a real POST to the existing /api/v3/enrich endpoint (or a new /api/v3/game/validate wrapper). Show actual Tanimoto similarity hits from ChEMBL/PubChem with compound names.
  • Victory: best molecule vs known drug comparison
    Add a side-by-side comparison: player's best molecule score vs the known drug score from game_levels.json. Show improvement delta: "You beat Paxlovid by 12%!" or "You matched Tamiflu!". For kids: "You found a molecule BETTER than the real medicine!"
  • Session statistics panel
    Victory screen shows: total attacks, best score achieved, score progression (simple bar chart using CSS widths), time taken. Fetched from /api/v3/game/session/<id> which already returns all attacks.
  • Display earned XP and badge
    The server already awards XP and badges on win — but the 3D game never shows them. Add the badge emoji and XP amount from the session's won response data. Animate badge drop with CSS keyframe.
  • Share your discovery
    A copy-to-clipboard button shares a text like: "I defeated COVID-19 Mpro in PathoHunt with a molecule scoring 0.76 — beating Paxlovid! 🏆 #PathoHunt #DrugDesign". Optional future: generate a shareable card image via canvas toDataURL.
